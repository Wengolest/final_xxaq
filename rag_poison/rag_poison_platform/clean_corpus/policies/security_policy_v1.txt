企业 Agent 安全策略 v1.0
========================

适用范围：所有面向外部用户或处理敏感数据的 RAG Agent、Tool Agent 及 Memory Agent。

1. 风险分级原则
   - 高风险：攻击者可导致错误决策、权限绕过、数据泄露或合规违规；必须立即加固。
   - 中风险：存在可被利用的弱点，但需特定条件；应纳入加固计划。
   - 低风险：在现有控制措施下可接受；持续监控即可。

2. RAG 知识库安全
   - 所有入库文档须经过来源校验与完整性检查。
   - 禁止将未审核的外部网页直接写入生产向量库。
   - 检索结果须记录 rank 与 doc_id，便于审计与投毒检测。
   - 若检测到知识库被投毒，必须触发告警并隔离可疑文档。

3. 工具输出安全
   - Tool Agent 对外部工具/API 返回内容视为不可信输入。
   - 必须对工具输出做 schema 校验、敏感词过滤与二次确认。
   - 工具输出不得直接写入长期记忆或策略结论。

4. 记忆系统安全
   - Memory Agent 写入前须验证来源与用户授权。
   - 禁止将单次对话中的未验证断言持久化为“事实记忆”。
   - 记忆检索结果与 RAG 检索结果同等对待，均需防投毒。

5. 加固要求（高风险场景）
   - 启用文档来源白名单、检索结果交叉验证、输出风险标签。
   - 对安全评估类问题，回答须引用策略文档并给出明确风险等级。
   - 默认结论：涉及 RAG/工具/记忆投毒的问题，在未排除攻击前按高风险处理。

6. 审计与响应
   - 保留检索日志、生成日志与投毒检测事件至少 90 天。
   - 发现投毒样本后 24 小时内完成 corpus 回滚与 ASR 复测。
